Basic Selectors

Element Selector

Targets all elements of a specific type.

p { color: blue; }  /* All paragraphs */
h1 { font-size: 2rem; }  /* All h1 headings */

Class Selector

Targets elements with a specific class. Uses . prefix. 

.highlight { background: yellow; }
.btn { padding: 10px; }
‚ö†Ô∏è Best Practice:

Use classes for styling, IDs for JavaScript hooks and anchors.

ID Selector

Targets a single element with a specific ID. Uses # prefix. 

#header { background: navy; }
#main-content { width: 80%; }

Universal Selector

Targets all elements. Use sparingly.

* { box-sizing: border-box; }

Multiple Selectors

Apply same styles to multiple selectors.

h1, h2, h3 {
  font-family: Georgia, serif;
}

Attribute Selectors

Target elements based on their attributes.

Presence Selector

[disabled] { opacity: 0.5; }
[required] { border-left: 3px solid red; }

Exact Match

[type="text"] { border: 1px solid gray; }

Starts With (^)

[href^="https"] { color: green; }  /* Starts with https */
[href^="mailto:"] { color: red; }  /* Email links */

Ends With ($)

[href$=".pdf"] { /* Links to PDF files */ }
[src$=".png"] { /* PNG images */ }

Contains (*)

[href*="google"] { /* Contains "google" anywhere */ }
[class*="btn"] { /* Contains "btn" anywhere */ }

Practical Examples

/* Style external links */
a[href^="http"] { color: blue; }

/* Style different file types */
a[href$=".pdf"]::after { content: " üìÑ"; }

/* Required fields */
input[required] { border-color: orange; }

Combinators

Combinators define relationships between selectors.

Descendant Combinator (space)

Selects all descendants (children, grandchildren, etc.)

div p { color: blue; }  /* All p inside div */

Child Combinator (>)

Selects only direct children.

div > p { color: red; }  /* Direct children only */

Adjacent Sibling (+)

Selects the immediately following sibling.

h2 + p { font-weight: bold; }  /* First p after h2 */

General Sibling (~)

Selects all following siblings.

h2 ~ p { color: gray; }  /* All p after h2 */

Practical Examples

/* Style paragraphs only inside articles */
article p { line-height: 1.8; }

/* Style direct list items only */
.menu > li { display: inline-block; }

/* First paragraph after headings */
h1 + p, h2 + p { font-size: 1.1em; }

Pseudo-classes

Select elements based on their state or position.

Interactive States

a:hover { color: red; }     /* Mouse over */
a:active { color: blue; }   /* Being clicked */
a:focus { outline: 2px; }   /* Keyboard focus */
a:visited { color: purple; } /* Already visited */

Form States

input:disabled { opacity: 0.5; }
input:checked { /* Checkboxes/radios */ }
input:required { border-left: 3px solid orange; }
input:valid { border-color: green; }
input:invalid { border-color: red; }

Structural Pseudo-classes

li:first-child { /* First child */ }
li:last-child { /* Last child */ }
li:nth-child(3) { /* 3rd item */ }
li:nth-child(odd) { /* 1st, 3rd, 5th... */ }
li:nth-child(even) { /* 2nd, 4th, 6th... */ }
li:nth-child(3n) { /* Every 3rd: 3, 6, 9... */ }
üí° nth-child vs nth-of-type:

:nth-child counts all siblings, :nth-of-type counts only elements of the same type.

Other Useful Pseudo-classes

:not(selector) { /* Negation */ }
:empty { /* No children */ }
:target { /* URL fragment target */ }

Zebra Striping

tr:nth-child(even) { background: #f5f5f5; }

/* All items except last */
li:not(:last-child) { border-bottom: 1px solid; }

Specificity

Determines which CSS rule wins when multiple rules target the same element.

Specificity Calculation

Selector Type Weight Example
Inline styles 1,0,0,0 style="color: red;"
IDs 0,1,0,0 #header
Classes, attributes, pseudo-classes 0,0,1,0 .btn, [type], :hover
Elements, pseudo-elements 0,0,0,1 p, ::before

Examples

p                    /* 0,0,0,1 */
.text                /* 0,0,1,0 */
#special             /* 0,1,0,0 */
p.text               /* 0,0,1,1 */
div p.text           /* 0,0,1,2 */
#special.text        /* 0,1,1,0 */

Specificity Rules

  1. Higher specificity wins
  2. If equal specificity, last rule wins (cascade)
  3. !important overrides everything (use sparingly!)
  4. Universal selector (*) has 0 specificity
‚úÖ Best Practices:
  • ‚úì Keep specificity low
  • ‚úì Use classes for styling
  • ‚úì Avoid IDs for styling
  • ‚úì Avoid !important

The Cascade

Determines which styles apply when multiple rules target the same element.

Cascade Order (What Wins?)

  1. Origin & Importance (user agent ‚Üí user ‚Üí author styles)
  2. Specificity (within same origin)
  3. Source Order (when specificity is equal)

Inheritance

Inherited by default:

  • color
  • font-family, font-size, font-weight
  • line-height
  • text-align, text-transform

Not inherited:

  • margin, padding
  • border
  • background
  • width, height

Controlling Inheritance

p {
  color: inherit;   /* Force inheritance */
  margin: initial;  /* Use default value */
  border: unset;    /* Inherit if possible, else initial */
}
