1. Content

The actual content of the element (text, images, etc.)

div {
  width: 200px;   /* Content width */
  height: 100px;  /* Content height */
}

2. Padding

Space between content and border. Inside the element. 

div {
  padding: 20px;  /* All sides */
  padding: 10px 20px;  /* Vertical | Horizontal */
  padding: 10px 20px 30px;  /* Top | H | Bottom */
  padding: 10px 20px 30px 40px;  /* T R B L (clockwise) */
  
  /* Individual sides */
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 30px;
  padding-left: 40px;
}
âœ… Padding Facts:
  • Background color/image extends into padding
  • Padding is transparent but shows background
  • Adds to total element size (by default)
  • Cannot be negative

3. Border

Line around the padding and content.

div {
  border: 2px solid black;  /* width | style | color */
  
  /* Individual properties */
  border-width: 2px;
  border-style: solid;  /* solid, dashed, dotted, etc. */
  border-color: black;
  border-radius: 8px;  /* Rounded corners */
  
  /* Individual sides */
  border-top: 1px solid red;
  border-right: 2px dashed blue;
}

4. Margin

Space outside the border. Separates elements from each other. 

div {
  margin: 20px;  /* All sides */
  margin: 10px 20px;  /* Vertical | Horizontal */
  margin: 0 auto;  /* Center block element */
  
  /* Individual sides */
  margin-top: 10px;
  margin-right: 20px;
  margin-bottom: 30px;
  margin-left: 40px;
}
âœ… Margin Facts:
  • Always transparent (no background)
  • Can be negative (overlapping)
  • Vertical margins collapse (see below)
  • Use margin: 0 auto to center

Box-Sizing Property

Controls how total width/height is calculated.

content-box (default)

Width/height = content only

box-sizing: content-box;

/* If width: 200px */
Total = 200px (content)
      + padding
      + border

border-box (recommended)

Width/height = content + padding + border

box-sizing: border-box;

/* If width: 200px */
Total = 200px (includes 
        content + padding 
        + border)
ğŸ’¡ Best Practice:

Always use border-box globally:

* {
  box-sizing: border-box;
}

Example Calculation

.box {
  width: 200px;
  padding: 20px;
  border: 5px solid black;
}

/* content-box: */
Total width = 200 + 40 (padding) + 10 (border) = 250px

/* border-box: */
Total width = 200px (everything included)

Display Properties

display: block

Takes full width available, starts on new line. Can set width/height. 

/* Examples: div, p, h1-h6, section, article, header, footer */
display: block;
  • âœ… Takes full width by default
  • âœ… Starts on new line
  • âœ… Can set width and height
  • âœ… Respects all margin/padding

display: inline

Flows with text, takes only content width. Cannot set width/height. 

/* Examples: span, a, strong, em, code */
display: inline;
  • âœ… Flows with text
  • âŒ Cannot set width/height
  • âš ï¸ Only respects left/right margin/padding
  • âš ï¸ Ignores top/bottom margin

display: inline-block

Best of both: flows inline but can set dimensions.

/* Great for: buttons, navigation items */
display: inline-block;
  • âœ… Flows inline (side by side)
  • âœ… Can set width and height
  • âœ… Respects all margin/padding

display: none

Removes element completely from layout (doesn't take up space).

display: none;  /* Element invisible and takes no space */

visibility: hidden

Hides element but keeps its space in layout.

visibility: hidden;  /* Invisible but keeps space */

Removes from layout flow

[Box1] [Box3]
/* Box2 gone */

visibility: hidden

Hides but keeps space

[Box1] [    ] [Box3]
/* Box2 hidden but space remains */

Width, Height & Sizing

Units

width: 300px;      /* Fixed pixels */
width: 50%;        /* Percentage of parent */
width: auto;       /* Default, fits content */
width: 100vw;      /* Viewport width */
width: 20rem;      /* Relative to root font-size */
width: 10em;       /* Relative to element font-size */

Min & Max Constraints

min-width: 200px;   /* Never smaller than 200px */
max-width: 1200px;  /* Never larger than 1200px */
min-height: 100vh;  /* At least full viewport height */
max-height: 500px;  /* Cap height at 500px */

Responsive Container Pattern

.container {
  max-width: 1200px;  /* Don't exceed 1200px */
  margin: 0 auto;     /* Center horizontally */
  padding: 0 20px;    /* Breathing room on sides */
}
ğŸ’¡ Pro Tip:

Use max-width instead of width for responsive containers. It allows elements to shrink on smaller screens.

Overflow

Controls what happens when content exceeds element's box.

Overflow Values

overflow: visible;  /* Default - content overflows */
overflow: hidden;   /* Clip overflow content */
overflow: scroll;   /* Always show scrollbars */
overflow: auto;     /* Scrollbars only when needed */

/* Individual axes */
overflow-x: auto;   /* Horizontal only */
overflow-y: scroll; /* Vertical only */

Common Patterns

/* Full height page */
.page {
  min-height: 100vh;
}

/* Scrollable chat area */
.chat {
  height: 400px;
  overflow-y: auto;
}

/* Horizontal scroll for wide tables */
.table-wrapper {
  overflow-x: auto;
}

Margin Collapse

When two vertical margins meet, they collapse to the larger margin (not the sum).

Example

.box1 {
  margin-bottom: 30px;
}

.box2 {
  margin-top: 20px;
}

/* Gap between boxes = 30px (NOT 50px!) */
/* The larger margin wins */

When Margins Collapse

  • âœ… Adjacent siblings (vertical only)
  • âœ… Parent and first/last child (if no padding/border)
  • âœ… Empty blocks

When Margins DON'T Collapse

  • âŒ Horizontal margins (left/right)
  • âŒ Elements with padding or border between them
  • âŒ Floated or absolutely positioned elements
  • âŒ Flexbox or Grid children
  • âŒ Inline-block elements
ğŸ’¡ Preventing Margin Collapse:
  • Add padding: 1px to parent
  • Add border: 1px solid transparent to parent
  • Use display: flex on parent
  • Use padding instead of margin
