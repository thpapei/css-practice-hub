What is CSS Grid?

CSS Grid is a two-dimensional layout system that allows you to create complex layouts with rows and columns simultaneously. Unlike Flexbox (one-dimensional), Grid gives you complete control over both axes.

üîë Key Concept:

Grid is two-dimensional, meaning it handles layout in rows AND columns at the same time.

Grid vs Flexbox

Use Grid When:

  • ‚úÖ Layout is 2D (rows AND columns)
  • ‚úÖ Need precise item placement
  • ‚úÖ Creating page layouts
  • ‚úÖ Defining structure first

Use Flexbox When:

  • ‚úÖ Layout is 1D (row OR column)
  • ‚úÖ Items size based on content
  • ‚úÖ Creating navbars, toolbars
  • ‚úÖ Items control their size
üí° Best Practice:

Use Grid for page layout, Flexbox for components. They work great together!

Grid Terminology

Grid Lines (vertical):    1   2   3   4
                          |   |   |   |
Grid Lines (horizontal): 1|---|---|---|  ‚Üê Grid Track (row)
                         2|---|---|---|
                         3|---|---|---|
                              ‚Üë
                         Grid Track (column)

Grid Cell = Single box at intersection
Grid Area = Multiple cells forming rectangle
  • Grid Container: Element with display: grid
  • Grid Items: Direct children of grid container
  • Grid Lines: Dividing lines (numbered from 1)
  • Grid Tracks: Space between two lines (row or column)
  • Grid Cells: Space between four lines
  • Grid Areas: One or more cells forming rectangle

Grid Container Properties

display

.container {
  display: grid;         /* Block-level grid */
  display: inline-grid;  /* Inline-level grid */
}

grid-template-columns & grid-template-rows

Define the column and row structure.

.container {
  /* 3 equal columns */
  grid-template-columns: 1fr 1fr 1fr;
  
  /* Shorthand with repeat() */
  grid-template-columns: repeat(3, 1fr);
  
  /* Mixed units */
  grid-template-columns: 200px 1fr 2fr;
  
  /* Auto-fit columns (responsive) */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  
  /* Define rows */
  grid-template-rows: 100px 200px auto;
}

Space between grid items.

  gap: 20px;          /* Equal row and column gap */
  gap: 20px 10px;     /* Row gap | Column gap */
  row-gap: 20px;      /* Only vertical gap */
  column-gap: 10px;   /* Only horizontal gap */

justify-items & align-items

Align items inside their grid cells.

.container {
  /* Horizontal alignment (justify) */
  justify-items: start;    /* Left align */
  justify-items: end;      /* Right align */
  justify-items: center;   /* Center */
  justify-items: stretch;  /* Default: fill cell */
  
  /* Vertical alignment (align) */
  align-items: start;      /* Top */
  align-items: end;        /* Bottom */
  align-items: center;     /* Center */
  align-items: stretch;    /* Default: fill cell */
}

justify-content & align-content

Align the entire grid within the container.

.container {
  /* Horizontal alignment of grid */
  justify-content: start;
  justify-content: end;
  justify-content: center;
  justify-content: space-between;
  justify-content: space-around;
  justify-content: space-evenly;
  
  /* Vertical alignment of grid */
  align-content: start;
  align-content: end;
  align-content: center;
  align-content: space-between;
}

grid-template-areas

Name grid areas for semantic layout.

.container {
  grid-template-columns: 1fr 3fr 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header header"
    "sidebar main aside"
    "footer footer footer";
}

/* Then assign items to areas */
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.aside   { grid-area: aside; }
.footer  { grid-area: footer; }

Grid Item Properties

grid-column & grid-row

Position items by grid lines.

.item {
  /* Start at line 1, end at line 3 (spans 2 columns) */
  grid-column: 1 / 3;
  
  /* Shorthand */
  grid-column-start: 1;
  grid-column-end: 3;
  
  /* Span syntax */
  grid-column: span 2;  /* Span 2 columns */
  
  /* Rows work the same way */
  grid-row: 1 / 3;      /* Span 2 rows */
  grid-row: span 2;
}

grid-area

Place item in named area OR use line numbers.

/* With template areas */
.item {
  grid-area: header;
}

/* With line numbers: row-start / col-start / row-end / col-end */
.item {
  grid-area: 1 / 1 / 3 / 4;
  /* Same as:
     grid-row: 1 / 3;
     grid-column: 1 / 4; */
}

justify-self & align-self

Override alignment for individual item.

.item {
  justify-self: start;   /* Left */
  justify-self: end;     /* Right */
  justify-self: center;  /* Center horizontally */
  justify-self: stretch; /* Fill width */
  
  align-self: start;     /* Top */
  align-self: end;       /* Bottom */
  align-self: center;    /* Center vertically */
  align-self: stretch;   /* Fill height */
}

Advanced Grid Examples & Patterns

grid-auto-flow (dense vs row)

Controls how implicit items are placed. dense attempts to fill holes. 

.container {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-auto-flow: row;        /* default - place by rows */
  grid-auto-flow: row dense;  /* try to fill gaps */
}

auto-fit vs auto-fill (responsive columns)

auto-fit collapses empty tracks; auto-fill preserves them. Use with minmax(). 

.grid {
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
}

Implicit grid & grid-auto-rows

Items placed beyond explicit tracks create implicit rows/columns controlled by grid-auto-rows / grid-auto-columns. 

.container {
  grid-template-columns: repeat(3,1fr);
  grid-auto-rows: 100px; /* newly created rows will be 100px */
}

Masonry-like layout (grid-auto-rows + row spans)

A common technique: set grid-auto-rows to a fixed row height and use grid-row: span N to vary heights. 

.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px,1fr));
  grid-auto-rows: 8px; /* small track size */
  gap: 10px;
}
.item { grid-row: span 25; /* 25 * 8px = 200px height approx */ }

Nested grids

Grid containers can be nested ‚Äî a grid item can itself be a grid container. 

.card { display: grid; grid-template-rows: auto 1fr auto; }
.card .meta { display: grid; grid-template-columns: auto 1fr; }

Named lines & line-based placement

You can name lines and place items by name: grid-column: header-start / header-end; 

.container {
  grid-template-columns: [start] 1fr [main-start] 2fr [end];
}
.item { grid-column: main-start / end; }

Subgrid (where supported)

subgrid allows a nested grid to inherit the parent's tracks. Support is improving ‚Äî check browser compatibility. 

.parent { display: grid; grid-template-columns: 1fr 2fr; }
.child { display: grid; grid-template-columns: subgrid; }

Best Practices & Performance

  • Prefer minmax() + auto-fit/auto-fill for responsive layouts.
  • Use small grid-auto-rows with span technique for masonry-like grids ‚Äî it's performant and avoids layout thrash.
  • Avoid overly deep nested grids when simple flex or a single grid suffices; nested grids are powerful but add complexity.
  • Test dense placement ‚Äî it can reorder items which may affect source order semantics (a11y).
  • When using subgrid, verify browser support for your target audience.

12-Column Grid System

.container {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: 1rem;
}

.col-6  { grid-column: span 6; }  /* Half width */
.col-4  { grid-column: span 4; }  /* Third width */
.col-12 { grid-column: span 12; } /* Full width */

Image Gallery

.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  grid-auto-rows: 200px;
  gap: 10px;

  /* Span random sizes for masonry effect */

Center Everything

  display: grid;
  place-items: center;  /* Shorthand for justify-items + align-items */
